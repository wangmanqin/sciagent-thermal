# SciTune：面向科学计算 Agent 的领域自适应微调与偏好对齐

## 项目定位

SciAgent 系统依赖 DeepSeek API（闭源大模型）作为推理后端。本项目探索：**能否通过领域自适应 LoRA 微调 + DPO 偏好对齐，让开源小模型（Qwen2.5-1.5B）在科学计算任务上逼近闭源大模型的表现？**

## 实验结果总览

| 阶段 | 耗时 | 关键指标 |
|------|------|---------|
| 数据生成 | ~30 min | 250 条 SFT + 129 对 DPO（DeepSeek API 知识蒸馏） |
| SFT 训练 | 9 min | Train Loss: 0.79→0.44, Val Loss: 0.60→0.52 |
| DPO 训练 | ~2 min | Train Acc: 80%→97.3%, Val Acc: 84.2%→94.7% |
| 三模型评测 | ~5 min | raw 45分 → SFT 50分 → **DPO 55分** |

**三模型对比评测（SciAgent Benchmark, 快速 3 题）：**

| 模型 | 平均分 | 通过率(≥60分) | 相对 raw 提升 |
|------|--------|-------------|-------------|
| raw (原始 Qwen2.5-1.5B-Instruct) | 45.0 | 0/3 (0%) | — |
| SFT (LoRA 微调后) | 50.0 | 0/3 (0%) | +5 |
| **DPO (SFT + DPO 对齐)** | **55.0** | **1/3 (33.3%)** | **+10** |


## Step 1-4: 原理验证阶段（GPT-2）

### Step 1: 看懂Transformer在干什么
**文件**: `step1_see_transformer.py`
- Tokenizer → Embedding → 12层Transformer Block → 自回归生成
- 可视化 Attention 权重，看到 softmax(QK^T/√d) 的实际输出

### Step 2: 全量微调
**文件**: `step2_finetune_gpt2.py`
- WikiText-2 数据 → tokenize → 128切块 → CrossEntropy loss
- loss: 4.19→3.38，训练8.5分钟

### Step 3: LoRA高效微调
**文件**: `step3_lora.py`
- W' = W + BA，r=8，只训练0.24%参数
- loss: 3.59，训练3.1分钟（全量的36%时间）

### Step 4: DPO偏好对齐
**文件**: `step4_dpo.py`
- **手写实现完整DPO算法**（非调库）
- Loss = -log σ(β·(log_ratio_chosen - log_ratio_rejected))
- 偏好正确率: 0% → 87.5%

## Step 5-7: 领域微调阶段（Qwen2.5-1.5B）

### 数据构建
**文件**: `build_dataset.py`
- 用 DeepSeek API 批量生成科学计算QA数据（知识蒸馏）
- 10个类别，28道种子题，每题扩展8个变体
- **最终数据量**: SFT 250条 + DPO 129对
- 覆盖领域：方程求解(36)、ODE(27)、优化(35)、统计(27)、信号处理(27)、曲线拟合(26)、线性代数(27)、数值方法(27)、PDE(2)、数据分析(16)
- 支持 `--resume`（追加模式）、`--sft-only`、`--dpo-only` 等参数

### Step 5: LoRA SFT on Qwen2.5-1.5B-Instruct
**文件**: `step5_sft_qwen.py`

| 技术选择 | 说明 |
|---------|------|
| Instruct 模型 | 优先使用 Qwen2.5-1.5B-Instruct，天然理解 ChatML 格式 |
| HF 镜像 | 自动设置 hf-mirror.com，解决国内网络问题 |
| 4bit NF4 量化 | 1.5B模型只占~1.5GB显存 |
| LoRA r=16, QKVO | 覆盖完整注意力机制四个投影矩阵，可训练参数占比 0.28% |
| 8bit AdamW | 优化器也压缩，配合4bit模型 |
| gradient_accumulation=2 | 等效batch_size=4 |
| EarlyStoppingCallback | patience=3，防止过拟合 |
| 数据量自适应 | 根据数据条数动态调整 epoch 和学习率 |
| Label Masking | 只在 assistant 回复部分计算 loss |

**实际训练结果**：
- 训练数据 225 条（90% 拆分），验证 25 条
- 3 epochs，171 steps，耗时 9 分钟
- Train Loss: 0.79 → 0.44（下降 44%）
- Val Loss: 0.60 → 0.52（稳步下降，未过拟合）
- 最佳 checkpoint: step 120, val_loss=0.519

### Step 6: DPO on Qwen2.5-1.5B-Instruct
**文件**: `step6_dpo_qwen.py`

**经历三轮调试，最终方案**：

| 轮次 | 方案 | 结果 |
|------|------|------|
| 第一轮 | Base 模型 + merge_and_unload + 无早停 | 生成崩溃（重复字符） |
| 第二轮 | Instruct 模型 + adapter 链式加载（add_adapter + set_adapter） | **loss 异常高（>10），acc=0%** |
| **第三轮** | **Instruct 模型 + SFT merge → DPO LoRA** | **loss=0.62→0.36, acc=97.3%** ✅ |

**第二轮失败的关键发现**：PEFT 的 `set_adapter("dpo")` 会**禁用** SFT adapter（`active_adapters` 从 `['default']` 变为 `['dpo']`），导致 policy 模型完全失去 SFT 能力，与 ref 模型的 log probability 差距巨大，DPO loss 飙到 11+，训练完全失效。

**最终方案**：先 merge SFT adapter 到 4bit 基座（接受微小舍入误差），再在 merged 模型上添加 DPO LoRA。ref 模型也做相同的 merge，确保起点一致。

**实际训练结果**：
- 训练数据 110 对（85% 拆分），验证 19 对
- 2 epochs，lr=5e-6，β=0.1，梯度裁剪 max_norm=1.0
- Train: loss 0.62→0.36, acc 80%→97.3%
- Val: loss 0.51→0.27, acc 84.2%→94.7%
- 偏好正确率: 62% → 69%（+7%）

- 三模型对比: 原始 Qwen2.5 vs SFT vs SFT+DPO

- 四维度评分（共100分）: 代码可执行性(30) + 代码完整性(30) + 输出相关性(20) + 回答结构(20)

## 关键技术发现

### 1. PEFT adapter 链式加载的陷阱
`add_adapter("dpo") + set_adapter("dpo")` 会禁用原有的 SFT adapter，导致模型完全丢失 SFT 能力。正确做法是先 `merge_and_unload()` SFT adapter，再用 `get_peft_model()` 添加新 LoRA。

### 2. DPO 对 SFT 质量的强依赖
DPO 是在已经很好的 SFT 模型上做"精修"，而非"补课"。SFT 基础不牢时，DPO 会加速退化。这与 Anthropic 等机构的实践一致。

### 3. Instruct vs Base 模型
使用 Instruct 模型作为基座比 Base 模型效果显著更好。Instruct 模型天然理解 ChatML 对话格式，SFT 起点更高，DPO 阶段更稳定。

### 4. 国内网络适配
HuggingFace 在国内不可达，通过在 import transformers 前设置 `os.environ["HF_ENDPOINT"] = "https://hf-mirror.com"` 解决。

## 技术栈

- **模型**: Qwen2.5-1.5B-Instruct, GPT-2
- **框架**: PyTorch, HuggingFace Transformers, PEFT
- **量化**: bitsandbytes (NF4 + 双重量化)
- **对齐**: 手写DPO实现（非TRL库）
- **评测**: 自研评测框架 + SciAgent Benchmark (10题, 微通道/传热等领域)
- **数据**: DeepSeek API 生成（知识蒸馏, 250 SFT + 129 DPO）
- **镜像**: hf-mirror.com（国内 HuggingFace 镜像）
